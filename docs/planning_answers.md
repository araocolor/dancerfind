## 📋 주제 1: 서비스 목적 & 타겟

### 1-1. 서비스 정의
- **댄스 클래스 플랫폼** — 강사가 클래스를 개설하고, 사용자가 검색하여 참여 신청하는 서비스
- **MVP 핵심 기능**: ① 클래스 개설 ② 클래스 검색 ③ 클래스 참여 신청

### 1-2. 타겟 사용자
- **모든 레벨 사용자** (입문자 ~ 전문가)
- **장르**: 살사 / 바차타 / 기타

### 1-3. 서비스 지역
- **국내(한국) 한정 / 한국어 전용**

### 1-4. 클래스 개설 권한
- **클래스 개설**: 일반 댄서 누구나 가능
- **이벤트/행사 개설**: 별도 회원 등급(권한)만 가능 → 회원 등급 시스템 필요

### 1-5. 수익 모델
- **완전 무료**
- 포인트/결제 시스템 **사용 안 함** (초기 논의 후 삭제 확정)

## 📋 주제 2: 요구사항 명세 & 정보구조

### 2-1. 회원 등급 (3단계)
| 등급 | 권한 |
|---|---|
| **일반회원** | 클래스 개설 / 참여 가능 |
| **프로** | 일반회원 권한 + 이벤트·행사 개설 가능 |
| **운영자(관리자)** | 모든 클래스/이벤트 개설·수정·삭제 권한 (전체 관리) |

### 2-2. 클래스 개설 필드
- 장르
- 제목
- 레벨
- 일시
- 장소
- 정원
- 설명본문
- 마감일
- 연락처
- 비용
- 강습이미지

### 2-3. 클래스 검색 흐름 (2단계)

**1단계 — 검색 조건 선택 페이지** (3가지 필터)
- 지역
- 모집 상태
- 클래스 구분 (1:1 강습 등)

**2단계 — 리스트 페이지**
- 1단계에서 선택한 조건으로 클래스 리스트 표시

**기본 옵션 저장 동작**
- 기본 옵션 설정이 **체크 상태** → 1단계 건너뛰고 바로 2단계(리스트)
- 기본 옵션 설정이 **체크 해제 / 없음** → 1단계(설정 페이지) 우선 표시
- → 사용자별 "기본 검색 조건" 저장 기능 필요

### 2-4. 클래스 참여 신청 흐름
1. 신청자: **[신청]** 버튼 클릭
2. 개설자: **승인 / 거절** 결정
3. 승인 시: 확정 + 신청자에게 **메시지 알림(수락 확인)** 전송
4. **취소 정책**:
   - 강습일 **1일 이상 남음** → 취소 버튼 활성화 (취소 가능)
   - 강습일 **1일 이내** → 취소 불가

### 2-5. 알림 방식
- **현재 단계는 웹페이지 개발 (앱 아님)**
- **기본**: 사이트 내 알림함(종 아이콘)에서 확인 — 외부 발송 없음
- **보조**: 카카오톡 푸시 (기술적으로 가능하면 추가)

## 📋 주제 7: 추가 기능 세부 사항

### 7-1. 홈(메인) 페이지
- 검색 창 (상단)
- 추천 클래스 리스트 (하단)

### 7-2. 클래스 리스트 정렬
- 사용자가 선택 가능: 최신순 / 마감임박순 / 인기순

### 7-3. 이벤트/행사 구조
- 별도 테이블 없음 — classes 테이블의 type 필드로 구분
- type 값: `class` (일반 클래스) / `event` (이벤트/행사)
- 이벤트 개설은 프로/운영자만 가능 (기존 2-1 정책 유지)

### 7-4. 프로 등급 승급
- 방식: 운영자가 관리자 페이지에서 수동으로 등급 변경
- 별도 신청/자동 승급 플로우 없음

### 7-5. 클래스 수정 정책
- 개설자는 모든 필드 수정 가능 (신청자 유무 무관)
- 수정된 클래스는 리스트/상세에서 "수정됨" 표시
- 승인된 신청자에게 수정 알림 전송
- 운영자 별도 알림 불필요

### 7-6. 신청자 관리 위치
- 클래스 상세 페이지 하단에 신청자 리스트 표시
- 개설자만 열람 가능
- 각 신청자 행에 승인 / 거절 버튼 제공

### 7-7. 마이페이지 구성
1. 내 프로필 편집 (닉네임, 사진, 자기소개, 활동지역)
2. 내가 개설한 클래스 목록
3. 내가 신청한 클래스 목록 (신청 상태 포함)
4. 알림 설정 (카카오톡 알림 수신 여부)

### 7-8. 클래스 키워드 검색
- 2단계 리스트 화면에서 텍스트 검색란 제공
- 검색 대상: 제목 + 설명본문
- 기존 지역/모집상태/구분 필터와 조합 가능

### 7-9. 클래스 상세 — 개설자 프로필 연동
- 상세 페이지에 개설자 닉네임 + 사진 표시
- 닉네임 클릭 → 개설자 회원 프로필 페이지로 이동

### 7-10. 회원 프로필 페이지 공개 정보
| 항목 | 공개 여부 |
|---|---|
| 닉네임 + 프로필 사진 | 공개 |
| 자기소개 (bio) | 공개 |
| 활동 지역 | 공개 |
| 이메일, 전화번호, 카카오ID | 비공개 |

## 📋 주제 8: Phase 0 기획 결정 사항 (12개 항목 확정)

### 8-1. 비로그인 접근 권한 범위
- **홈 / 클래스 검색 1단계·2단계 / 클래스 상세 / 회원 프로필** → 비로그인 접근 허용
- **클래스 신청** → 로그인 필요 (신청 버튼 클릭 시 로그인 페이지로 유도)
- middleware.ts 보호 경로: `/classes/new`, `/mypage`, `/notifications`, `/admin`, `/onboarding`

### 8-2. 지역 필터 옵션 목록
- 고정 커스텀 목록 (댄스 커뮤니티 주요 지역 기준)
  ```
  전체 / 서울강남 / 서울강북 / 경기도 / 부산 / 대구 / 대전 / 광주 / 창원 / 전주 / 제주
  ```
- 정적 상수 배열로 관리 (코드 테이블 별도 DB 불필요)

### 8-3. 홈 추천 클래스 알고리즘
- **로그인 사용자**: 내 활동지역 + 선호 장르(최근 신청 클래스 기반) → 마감임박순
- **비로그인 사용자**: 전체 인기순 (신청자 수 기준)
- 표시 구성: 장르별 섹션 없이 단일 리스트로 표시

### 8-4. 인기순 정렬 기준
- **조회수 (view_count)** 기준
- `classes` 테이블에 `view_count INTEGER NOT NULL DEFAULT 0` 필드 추가 필요
- 클래스 상세 페이지 진입 시 +1 (중복 방지는 MVP 이후 적용)

### 8-5. 클래스 삭제 정책
- **삭제 기능 없음** — 상태를 '취소'로만 변경 가능
- **단, 신청자가 0명인 경우** → 즉시 삭제 허용
- **신청자가 있는 경우** → 삭제 불가, 모든 신청자가 취소하면 자동 삭제 처리
- 소프트 삭제 불필요 (신청자 0명 상태에서만 하드 삭제)

### 8-6. 카카오맵 UI 패턴
- **클래스 개설 시**: 지도 아이콘 클릭 → 카카오맵 주소 검색 패널 팝업 → 선택 시 위도/경도 자동 저장
- **클래스 상세 페이지**: 카카오맵 지도 위젯으로 위치 표시 (주소 텍스트 + 지도 병행)

### 8-7. 카카오 알림톡 발송 시나리오 (4가지)
| 시점 | 수신자 |
|---|---|
| 신청 수령 시 | 개설자 |
| 승인 완료 시 | 신청자 |
| 클래스 수정 시 | 승인된 신청자 전원 |
| 강습 D-1 리마인더 | 승인된 신청자 전원 |

- 거절 시는 사이트 내 알림함으로만 처리 (알림톡 미발송)

### 8-8. 회원가입 온보딩
- **온보딩 단계 있음**
- 카카오 로그인 최초 진입 → `/onboarding` 페이지 → 닉네임 + 활동지역 입력 → 홈 이동
- 닉네임 미입력 시 서비스 이용 제한 (신청/개설 불가)

### 8-9. 알림함 UI 형태
- 헤더 종 아이콘 클릭 → `/notifications` 별도 페이지 이동
- 읽지 않은 알림 수 배지 표시 (종 아이콘 우측 상단)

## 📋 주제 9: 추가 결정 사항 (Q1~Q15)

### 9-1. 검색 리스트 방식
- **무한 스크롤** / 20개씩 표시 / 다음 20개 미리 prefetch (캐싱)
- 구현: React Query `useInfiniteQuery` + `prefetchInfiniteQuery`

### 9-2. 기본 검색 조건 저장 위치
- **로그인 사용자** → profiles 테이블
- **비로그인 사용자** → sessionStorage (탭 닫으면 초기화)

### 9-3. 클래스 레벨 옵션
- `입문 / 초급 / 중급 / 고급 / 올레벨`

### 9-4. 닉네임 중복
- **중복 불허** — DB `UNIQUE` 제약 / 온보딩·프로필 수정 시 실시간 중복 확인 API 필요

### 9-5. 개설자 클래스 취소 시 알림
- 수단: **알림톡 + 사이트 내 알림** 모두 발송
- 대상: 승인된 신청자 전원
- 삭제 정책 재확인: 신청자 있으면 하드 삭제 불가 → 취소 상태로만 변경 → 신청자 전원 취소 시 자동 삭제

### 9-6. 프로 등급 신청
- 마이페이지에 **"프로 신청" 버튼** → 관리자 페이지에서 검토/승인
- 관리자 페이지 기능 추가: 프로 신청 목록 조회 + 승인/거절

### 9-7. D-1 리마인더
- **MVP 제외** — 서비스 안정화 후 Vercel Cron으로 추가

### 9-8. 이미지 없는 클래스 썸네일
- **목록형**: 개설자 아바타만 표시
- **카드형**: 강습이미지 480px 표시
- 강습이미지 없을 시: 장르별 기본 이미지 3장 자동 적용 (살사/바차타/기타·페스티벌·이벤트)
- 기본 이미지 파일은 별도 제공 예정
- 이미지 압축 스펙: 아이콘 200px / 카드용 480px / 풀사이즈 1034px

### 9-9. 회원 프로필 클래스 목록
- **모집중인 클래스만** 표시

### 9-10. 1:1 클래스 정원
- 개설자 직접 입력 가능 — "1:1"은 강습 방식 라벨이며 정원과 무관

### 9-11. Application 상태값 및 거절 처리
- 상태값: `신청대기 / 승인 / 취소` (거절 상태 없음)
- 미승인 = 암묵적 거절 (감정적 부담 최소화 의도)
- **마감 D-1 미승인 신청 → 자동 취소 처리** (Vercel Cron MVP 포함)

### 9-12. 장르 옵션 (확정)
- `살사 / 바차타 / 페스티벌 / 이벤트 / 기타`
- `classes.type` 필드(`class / event`)와 `genre` 필드는 독립적으로 유지
  - `type`: 개설 권한 구분 (class = 일반회원 / event = 프로·운영자)
  - `genre`: 콘텐츠 분류

### 9-13. 장르 검색 필터
- 검색 2단계 리스트 화면에 **장르 칩 필터** 추가
- 검색 1단계 필터: 지역 / 모집상태 / 클래스구분 / 장르 (4개로 확장 또는 2단계 칩으로만)

### 9-14. 실시간 알림
