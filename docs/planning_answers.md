## 📋 주제 1: 서비스 목적 & 타겟

### 1-1. 서비스 정의
- **댄스 클래스 플랫폼** — 강사가 클래스를 개설하고, 사용자가 검색하여 참여 신청하는 서비스
- **MVP 핵심 기능**: ① 클래스 개설 ② 클래스 검색 ③ 클래스 참여 신청

### 1-2. 타겟 사용자
- **모든 레벨 사용자** (입문자 ~ 전문가)
- **장르**: 살사 / 바차타 / 기타

### 1-3. 서비스 지역
- **국내(한국) 한정 / 한국어 전용**

### 1-4. 클래스 개설 권한
- **클래스 개설**: 일반 댄서 누구나 가능
- **이벤트/행사 개설**: 별도 회원 등급(권한)만 가능 → 회원 등급 시스템 필요

### 1-5. 수익 모델
- **완전 무료**
- 포인트/결제 시스템 **사용 안 함** (초기 논의 후 삭제 확정)

## 📋 주제 2: 요구사항 명세 & 정보구조

### 2-1. 회원 등급 (3단계)
| 등급 | 권한 |
|---|---|
| **일반회원** | 클래스 개설 / 참여 가능 |
| **프로** | 일반회원 권한 + 이벤트·행사 개설 가능 |
| **운영자(관리자)** | 모든 클래스/이벤트 개설·수정·삭제 권한 (전체 관리) |

### 2-2. 클래스 개설 필드
- 장르
- 제목
- 레벨
- 일시
- 장소
- 정원
- 설명본문
- 마감일
- 연락처
- 비용
- 강습이미지

### 2-3. 클래스 검색 흐름 (2단계)

**1단계 — 검색 조건 선택 페이지** (3가지 필터)
- 지역
- 모집 상태
- 클래스 구분 (1:1 강습 등)

**2단계 — 리스트 페이지**
- 1단계에서 선택한 조건으로 클래스 리스트 표시

**기본 옵션 저장 동작**
- 기본 옵션 설정이 **체크 상태** → 1단계 건너뛰고 바로 2단계(리스트)
- 기본 옵션 설정이 **체크 해제 / 없음** → 1단계(설정 페이지) 우선 표시
- → 사용자별 "기본 검색 조건" 저장 기능 필요

### 2-4. 클래스 참여 신청 흐름
1. 신청자: **[신청]** 버튼 클릭
2. 개설자: **승인 / 거절** 결정
3. 승인 시: 확정 + 신청자에게 **메시지 알림(수락 확인)** 전송
4. **취소 정책**:
   - 강습일 **1일 이상 남음** → 취소 버튼 활성화 (취소 가능)
   - 강습일 **1일 이내** → 취소 불가

### 2-5. 알림 방식
- **현재 단계는 웹페이지 개발 (앱 아님)**
- **기본**: 사이트 내 알림함(종 아이콘)에서 확인 — 외부 발송 없음
- **보조**: 카카오톡 푸시 (기술적으로 가능하면 추가)

## 📋 주제 3: 기술 스택

### 3-1. 프론트엔드 프레임워크
- **Next.js (App Router)** — SSR/SSG, SEO 유리, 최신 권장 패턴

### 3-2. 백엔드 구성
- **Next.js API Routes + Supabase** — Next.js가 프론트/API 담당, Supabase가 DB/인증/스토리지 담당

### 3-3. 데이터베이스
- **PostgreSQL** — Supabase 기반 (자동 적용)

### 3-4. 외부 연동
- **카카오 중심** — 카카오맵 API + 카카오 알림톡 + 카카오 소셜 로그인

### 3-5. 회원 인증
- **카카오 소셜 로그인 단독** — 이메일 옵션 없음
